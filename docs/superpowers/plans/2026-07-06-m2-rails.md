# M2-Rails Implementation Plan — Submission Through the Pipes

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the three ledgered M2 blockers, then send the first real cumulative quarterly update into HMRC's sandbox end-to-end: local records → derived totals → api PUT → immutable receipt → dashboard shows it. Plus HMRC's own calculation displayed beside our estimate.

**Architecture:** Records STAY local-first in the browser (privacy differentiator: individual records never leave the device — HMRC only ever receives category TOTALS, which is exactly what the cumulative-update API takes). The api gains ITSA submission endpoints that accept derived totals, PUT them to HMRC, and store an immutable receipt (existing receipts pattern). Wire shapes come from HMRC's real OAS on GitHub — fetch before coding, never guess (the dash-Task-1 discipline).

**Tech Stack:** existing (Hono api, engine, Next static front). No new deps.

## Global Constraints

- Sandbox-only guards on ALL new ITSA endpoints (no_such_door pattern). Copy discipline: submission UI says "Send to HMRC (sandbox demo)" — never implies production filing.
- Money: integer pence in engine/front; HMRC wire wants decimal pounds — conversion at the api boundary ONLY, with round-trip tests (123456p ↔ 1234.56).
- Wire field names verified against the live OAS (github.com/hmrc/self-employment-business-api v5.0, property-business-api v6.0, individual-calculations-api v8.0, business-details-api v2.0) — every implementer fetches the schema for the endpoint they touch and records the URL in their report.
- Receipts immutable: unique(entity, rail, tax_year, quarter_end) with 23505 race recovery (existing VAT pattern generalised).
- All suites green (engine 128, frontend 148, api 31 + new); typecheck; existing tests unmodified. Commits end with: `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>`.

### Task 2: Business discovery + submission endpoint (api)

**Files:**
- Create: `api/src/routes/itsa-submit.ts` (mounted under /v1/itsa), `api/migrations/004_itsa_receipts.sql`
- Test: `api/src/routes/__tests__/itsa-submit.test.ts`

**Interfaces (frontend Task 4 depends on):**
- `GET /v1/itsa/:id/businesses` → `{ businesses: [{ businessId, typeOfBusiness, tradingName? }] }` via Business Details API v2.0 list-businesses (fetch OAS for exact path/shape; Accept 2.0; Gov-Test-Scenario passthrough).
- `POST /v1/itsa/:id/quarterly-update` body `{ taxYear: '2026-27', businessId, typeOfBusiness: 'self-employment' | 'uk-property', quarterIndex: 1|2|3|4, election: 'standard'|'calendar', totals: Record<string, number> /* PENCE */ }` → api converts pence→pounds (2dp), maps totals onto the cumulative period summary payload per the OAS (SE: periodIncome {turnover, other}, periodExpenses {...}; property v6 equivalent — FETCH THE OAS, the research flagged v6 field names as needing verification), computes periodStartDate/periodEndDate from engine quarters (`@taxsorted/engine/uk/itsa` is importable from the api workspace — engine is a workspace dep; if api/package.json lacks it, add it), PUTs to HMRC, stores receipt, returns `{ receipt: { id, taxYear, quarterIndex, periodEnd, submittedAt, hmrcCorrelationId? } }`.
- `GET /v1/itsa/:id/receipts` → `{ receipts: [...] }` (list, newest first).
- Idempotency: resubmission same (entity, rail='itsa', taxYear, quarter periodEnd) UPDATES the receipt (cumulative resend is the MTD correction model) — receipt keeps `supersededCount`.
- Validation: totals keys validated against engine `categoriesFor(typeOfBusiness…)` wire keys; unknown key → 422 with the key named; negative/non-integer pence → 422.

- [ ] Steps: RED (mocked-HMRC tests: happy path SE, property, unknown-category 422, pence→pounds conversion exactness, resubmission updates not duplicates, sandbox guard) → fetch OAS + implement → GREEN → commit `feat(api): ITSA quarterly submission — cumulative PUT + receipts`.
