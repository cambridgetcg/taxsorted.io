# M1 — Prep-Tool Front (“Quarter One, sorted”) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the free prep experience before 7 Aug 2026: eligibility checker, local-first record keeper with CSV import, quarterly-figures calculator, liability estimator, mileage tool, de-panic page and refreshed Learn content — all education-labelled, no HMRC submission.

**Architecture:** Extend the live `frontend/` workspace (Next.js App Router, `output:'export'`, Tailwind 4 paper/ink/accent tokens, Radix-style `components/ui/*`). All tax math comes from `@taxsorted/engine/uk/itsa` (plan M1-engine) — **no figures in JSX, ever** (that's how the 45p-mileage class of error happens). Records live client-side only in M1 (IndexedDB, local-first — no accounts until M2), with JSON/CSV export. Every tool page carries the education disclaimer and cites its numbers via the engine's `Cited` metadata.

**Tech Stack:** Next.js 16 static export, React 19, Tailwind 4, vitest 4 (workspace `frontend`), `idb-keyval` (~600B, the one new dep) for IndexedDB.

## Global Constraints

- Depends on plan M1-engine being complete (`@taxsorted/engine/uk/itsa` surface, exact names in its Task 9).
- Static-export constraints: no middleware/route handlers/server actions; new routes need no dynamic segments (all static paths).
- Copy rules: never "HMRC approved" (only "recognised", and we are NOT yet — say "working towards HMRC recognition"); disclaimer on every tool: education not advice; penalty messaging must always pair "no quarterly penalty points in 2026-27" with "late-payment penalties and the 31 January return are still real".
- Accessibility from day one: semantic landmarks, labels on every input, visible focus, keyboard path through every flow (full WCAG 2.1 AA audit is an M3 gate, but nothing here may knowingly fail it).
- All existing frontend tests (12) stay green. Run: `npm test --workspace frontend`. Build check: `npm run build --workspace frontend`.

```ts
export interface CsvMapping { date: string; amount: string; description?: string } // header names in the user's file
export function parseCsv(text: string): { headers: string[]; rows: string[][] }     // RFC 4180: quotes, escaped quotes, CRLF
export function guessMapping(headers: string[]): CsvMapping | null                  // date/amount/desc header heuristics
export function suggestCategory(description: string, source: SourceType): string    // keyword → category key, default otherExpenses/other
export function toRecords(parsed: ReturnType<typeof parseCsv>, mapping: CsvMapping, source: SourceType):
  Array<{ record: Omit<LedgerRecord, 'id'>; guessed: boolean; warning?: string }>
// amount sign decides kind (negative → expense); dd/mm/yyyy and yyyy-mm-dd both accepted; bad rows → warning, not throw
```
